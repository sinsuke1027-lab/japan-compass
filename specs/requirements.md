# 要件定義書

## プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | Japan Compass |
| 目的 | 海外旅行者が日本滞在中に感じる文化・マナー・サステナブル観光に関する不安を解消する |
| 対象ユーザー | 日本を訪れる外国人旅行者（英語話者を中心とした訪日観光客） |
| 作成日 | 2026-05-12 |
| 最終更新 | 2026-06-14 |

---

## 1. 背景・目的

### 1.1 背景

訪日外国人旅行者は年々増加しているが、電車内のマナー・神社仏閣での作法・ゴミの分別など、日本独自の文化ルールを知らないまま来日するケースが多い。また「お賽銭」「地元への貢献」といったサステナブルな観光活動への関心も高まっているが、やり方がわからず機会を逃している旅行者も多い。

### 1.2 目的

- 日本の文化・マナーをシーン別に英語でわかりやすく提供する
- 神社参拝・お賽銭・地域貢献型体験など、サステナブルな観光活動を旅行者が実践しやすくする
- 緊急時・困りごとへの対処法を即座に参照できるようにする

### 1.3 解決する課題

| 現状の問題 | 解決後の状態 |
|---|---|
| 電車でのマナーがわからず周囲に迷惑をかけてしまう | シーン別マナーガイドで事前・その場で確認できる |
| 神社のお作法・お賽銭の意味がわからない | 参拝ガイドと意味の解説で文化的背景まで理解できる |
| 緊急時（救急・警察・災害）にどう動けばよいかわからない | 緊急連絡先と行動手順を即座に参照できる |
| サステナブルな観光がしたいが具体的な方法がわからない | エコ活動・地域支援スポットをリストアップして誘導できる |

## 2. 対象ユーザー・ステークホルダー

| ユーザー種別 | 説明 | 主な利用シーン |
|---|---|---|
| 訪日旅行者（メイン） | 日本語が読めない英語話者。スマホ依存度が高い | 観光地・移動中・食事中・緊急時 |
| 長期滞在者 | ワーキングホリデー・留学生など数ヶ月滞在 | 日常生活のマナー確認、緊急時対応 |
| アプリ管理者 | コンテンツ更新担当（将来的に） | Supabase ダッシュボードからコンテンツ管理 |

## 4. 非機能要件

### 4.1 パフォーマンス
- 画面初期表示: 2秒以内
- コンテンツはオフラインキャッシュ対応（機内・電波弱い環境でも基本機能が使える）
- 地図表示: MapboxまたはGoogle Maps SDK使用

### 4.2 セキュリティ
- 認証方式: 匿名認証（Supabase Anonymous Auth）でサインアップ不要で使える
- 通信暗号化: HTTPS/TLS 必須
- 個人情報: 位置情報は緊急時・マップ機能のみ取得、不使用時は非取得

### 4.3 可用性
- Supabase Free/Pro プランの範囲内での運用
- コンテンツデータは CDN キャッシュで冗長化

### 4.4 スケーラビリティ
- MVP: 月間 1,000 DAU を想定
- Supabase の Row Level Security でデータアクセス制御

## 6. 制約条件

- 技術的制約: MVP は iOS / Android 両対応（Expo Go で動作確認）
- 予算的制約: Supabase Free プランで MVP を完結させる（月 500MB DB / 1GB Storage 以内）
- スケジュール制約: テンプレート検証目的のため、アプリ品質よりプロセス・CI フローの検証を優先
- 法的制約: 位置情報の取得はユーザーの明示的な許可が必要（iOS/Android ガイドライン遵守）
- 言語: MVP は英語のみ。日本語 UI は不要

## 8. 用語定義

（`docs/glossary.md` を参照）

主要用語:
- **Osaisenl (お賽銭)**: Coin offerings at Shinto shrines or Buddhist temples
- **Sustainable tourism**: 地域文化・環境への負荷を最小化しながら地域に貢献する観光スタイル
- **DAU**: Daily Active Users
