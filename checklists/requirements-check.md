# 要件定義 完了チェックリスト

要件定義フェーズの完了条件を確認します。
全項目にチェックが入るまで、基本設計フェーズへ進まないでください。

---

## 基本項目

### プロジェクト概要
- [ ] プロジェクトの目的が明確に記述されている
- [ ] 解決する課題が具体的に定義されている
- [ ] 対象ユーザーが特定されている

### 機能要件
- [ ] Must/Should/Couldで優先度が分類されている
- [ ] 各機能にIDが付番されている
- [ ] 機能の説明が「何をするか」で書かれている（「どうやるか」ではない）

### 非機能要件
- [ ] パフォーマンス要件が数値で定義されている
- [ ] セキュリティ要件が記載されている
- [ ] 可用性目標が定義されている

### スコープ
- [ ] スコープ外（やらないこと）が明示されている
- [ ] スケジュール制約が記載されている

---

## AI検査項目（Claude Codeが確認）

- [ ] 矛盾する要件がないか
- [ ] 実現不可能な要件がないか
- [ ] セキュリティ観点での懸念事項はないか
- [ ] 用語が`docs/glossary.md`と一致しているか
- [ ] 要件から想定されるIssueは全て起票済みか
