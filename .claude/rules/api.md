# API開発ルール

このファイルはAPIのコードを編集するときのみ有効です。

## エンドポイント設計

- RESTful原則に従う（`specs/api-spec.yaml`の定義に沿う）
- URLはケバブケース（例: `/user-profiles`）
- バージョニング必須（例: `/api/v1/`）
- HTTPメソッドを正しく使う（GET/POST/PUT/PATCH/DELETE）

## レスポンス形式

```json
// 成功時
{ "data": {...}, "meta": {...} }

// エラー時
{ "code": "ERROR_CODE", "message": "ユーザー向けメッセージ" }
```

- エラーコードは定数として定義し、ハードコードしない
- エラーメッセージには「何が起きたか」と「次にどうすればよいか」を含める

## セキュリティ

- 全エンドポイントに認証チェックを実装（明示的に除外するもの以外）
- リクエストボディは必ずバリデーション
- SQLは必ずプリペアドステートメントを使用
- レスポンスに内部情報（スタックトレース・DB構造）を含めない

## 禁止事項

- `SELECT *` の使用（必要なカラムを明示する）
- N+1クエリ（ループ内でのDB呼び出し）
- ユーザー入力を直接SQLに埋め込む

## ログ

- リクエスト/レスポンスのログを記録（個人情報はマスク）
- エラーはCloud Loggingに送信
- パスワード・トークンをログに含めない
