# スキル: テンプレートへのフィードバック反映

## 使い方
```
/sync-template
```

## 概要
`setup-templete/TEMPLATE_FEEDBACK.md` に蓄積された「未反映」のフィードバックを読み取り、
`project-template/` の対象ファイルに反映する。反映後はステータスを `✅ 反映済み` に更新する。

---

## 処理手順

### STEP 1: フィードバックを読み込む
`../../TEMPLATE_FEEDBACK.md`（= `setup-templete/TEMPLATE_FEEDBACK.md`）を読み込む。
ステータスが `⬜ 未反映` の行だけを抽出し、件数と内容を表示する。

未反映がなければ以下を表示して終了する:
```
✅ 反映すべきフィードバックはありません。
```

### STEP 2: 内容を確認してユーザーに承認を求める

抽出した未反映フィードバックを以下の形式で一覧表示する:

```
## 反映予定のフィードバック（X件）

| # | 対象ファイル | 提案内容 |
|---|------------|---------|
| 1 | ...        | ...     |
```

「これを project-template に反映してよいですか？」と確認してから次へ進む。

### STEP 3: project-template へ反映する

承認されたフィードバックを順番に処理する。

各フィードバックについて:
1. `../../project-template/` の対象ファイルを読み込む
2. 提案内容に基づいて編集する
3. 変更内容を1行で報告する

### STEP 4: TEMPLATE_FEEDBACK.md のステータスを更新する

反映したフィードバックのステータスを `⬜ 未反映` → `✅ 反映済み` に変更する。

反映ログに以下を追記する:
```
| {今日の日付} | {反映件数} | {内容サマリー} |
```

### STEP 5: 完了報告

```
## ✅ テンプレート反映完了

反映件数: X件
対象ファイル: （変更したファイルの一覧）

TEMPLATE_FEEDBACK.md のステータスを更新しました。
```

---

## 注意事項
- 承認なしに project-template を編集しない（STEP 2 の確認は省略不可）
- 既存の project-template のルール・構造を壊さない範囲で反映する
- 判断が難しいフィードバックはスキップしてユーザーに判断を仰ぐ
