# スキル: セッション開始時の状況確認

## 使い方
```
/session-start
```

## 概要
セッション開始時に `MEMORY.md` を読んで前回の続きを即座に把握する。
毎回「MEMORY.md を読んで」と指示しなくて済むようにするスキル。

---

## 処理手順

### STEP 1: MEMORY.md を読む
`MEMORY.md` を読んで現在の状態を把握する。

> **MEMORY.md がテンプレートのまま（未更新）の場合:**
> `最終更新: YYYY-MM-DD` のままであれば初回セッションと判断し、
> 以下のメッセージを出力して終了する:
> 「MEMORY.md がまだ初期状態です。`/workflow-new-project` を実行してプロジェクトを開始してください。」

### STEP 2: 状況サマリーを出力する
以下のフォーマットで報告する:

```
## 📋 セッション開始

**フェーズ:** {現在のフェーズ}
**最終更新:** {日時}
**作業ブランチ:** {ブランチ名 or "-"（未設定の場合）}

### 前回完了したこと
{✅ 直近の完了タスク から上位 3 件}

### 今日やること（未完了タスク）
{🔄 作業中・未完了タスク の一覧}
未完了タスクがない場合は「積み残しなし」と表示する。

### 注意事項
{⚠️ 重要な決定事項・🐛 既知の問題 から要点のみ。なければ省略}
```

### STEP 3: 次のアクションを提案する
未完了タスクの中から「今日着手すべきもの」を 1〜2 件提案する。
判断基準:
- Issue# が若いもの（古い未解決を優先）
- 状態が「中断」のもの（再開ポイントが明確なもの）
- 前回の完了タスクと関連するもの（文脈の継続）

提案フォーマット:
```
### 💡 おすすめの着手順
1. {Issue# or タスク名} — {理由を一言}
2. {Issue# or タスク名} — {理由を一言}

準備ができたら「続きをやって」と指示するか、別のタスクを指定してください。
```

---

## 注意事項
- MEMORY.md を読むだけで、ファイルを更新しない
- セッション終了時は `/update-memory` を実行すること
- 作業中ブランチが記載されている場合は `git branch` で実際の状態と一致するか確認してもよい
