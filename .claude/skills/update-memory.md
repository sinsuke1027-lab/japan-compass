# スキル: セッション終了時のMEMORY.md更新

## 使い方
```
/update-memory
```

または特定のフェーズを指定:
```
/update-memory フェーズ: 実装中
```

## 概要
セッション内で行った作業を `MEMORY.md` に標準フォーマットで記録する。
これを習慣にすることで、次のセッションで「どこまでやったか」を
Claude Code が即座に把握できるようになる。

---

## 処理手順

### STEP 1: 今日の作業を棚卸しする
今回のセッションで完了したこと・中断したことを会話履歴から整理する。

確認する観点:
- 完了したタスク（コミット・PR作成・マージ等）
- 中断した作業（次回の続きになるもの）
- 判断・決定した事項（技術選定・設計方針等）
- 発見した問題・負債

### STEP 2: MEMORY.md の現在の内容を読み込む
`MEMORY.md` を読んで、更新すべき箇所を特定する。

> **初回セッションの場合（MEMORY.md がテンプレートのまま）:**
> `最終更新: YYYY-MM-DD` のままであれば初期テンプレートと判断し、
> このSTEPはスキップして、STEP 3でテンプレートを上書きする形で新規作成する。
> 既存の「更新ログ」や「完了タスク」はすべて削除して今回の内容のみ記載する。

### STEP 3: MEMORY.md を更新する
以下の各セクションを更新する。

#### 更新するセクション

**📍 現在の状態**
```
最終更新: {今日の日付 YYYY-MM-DD HH:MM}
現在のフェーズ: {以下から選択}
  未開始 / 要件定義 / 基本設計 / 実装 / テスト / デプロイ
作業中ブランチ: {現在のブランチ名 or -}
```

**✅ 直近の完了タスク** （上に新しいものを追記）
```
- {今日の日付}: {完了した内容}（#{Issue番号}）
```

**🔄 作業中・未完了タスク**
完了したタスクを削除し、新しい未完了タスクを追加する。
```
| Issue# | 内容 | 状態 | メモ |
| #XX | ... | 中断 | {次回の再開ポイント} |
```

**⚠️ 重要な決定事項・注意点**
今日決定した技術的な判断・制約を追記する（既存の重要事項は維持）。

**🐛 既知の問題・負債**
新たに発見した問題を追記する。既に対処済みのものは削除する。

**📝 セッション更新ログ** （上に新しいものを追記）
```
| {今日の日付 YYYY-MM-DD} | {作業内容の1行サマリー} | {成功/課題あり} |
```

### STEP 4: 更新完了を報告する
```
## 📝 MEMORY.md 更新完了

更新内容:
- 完了タスク: X件追記
- 未完了タスク: X件更新
- 現在フェーズ: {フェーズ名}

次回セッション開始時: 「MEMORY.mdを読んで前回の続きから再開して」と指示してください。
```

---

## 注意事項
- 機密情報（APIキー・パスワード）を MEMORY.md に書かない
- 「なんとなく覚えている」ではなく、具体的なファイル名・Issue番号を記録する
- セッション終了前に必ず実行するよう習慣化する
