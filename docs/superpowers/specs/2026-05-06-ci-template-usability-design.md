# 設計書: CIワークフロー テンプレート使いやすさ改善

- **作成日:** 2026-05-06
- **対象:** フェーズ1 — テスト自動実行 CI ワークフロー完成
- **ステータス:** 承認済み

---

## 背景・目的

`ci.yml` は骨格のみ存在し、全ての技術スタック別ステップがコメントアウトされた状態。
テンプレートを使い始める人（GitHub Actions をコピペレベルで使えるユーザー）が
「どこを触れば動くか分からない」「CIが通ったか落ちたかPRで分かりにくい」という
つまずきを解消することを目的とする。

---

## 解決する課題

| 課題 | 現状 | 解決後 |
|------|------|--------|
| セットアップ手順が不明瞭 | ファイルを開いてもどこを編集すればいいか分からない | バナーコメントと視覚的マーカーで迷わず設定できる |
| CI結果がPRで分かりにくい | デフォルトの緑/赤バッジのみ | 成功・失敗の詳細がPRコメントとして自動投稿される |

---

## 変更対象

### 1. `.github/workflows/ci.yml` の改善

#### 1-1. セットアップバナーの追加

ファイル冒頭に以下の案内コメントを追加する:

```yaml
# ============================================================
# 🚀 セットアップ手順（プロジェクト開始時に一度だけ設定）
#   STEP 1: 使用するスタックを下の「選択スタック」に記入する
#   STEP 2: 対応するブロックのコメントを解除する（▼〜▲ の範囲）
#   STEP 3: PRを作成してCIが動くことを確認する
#
# 選択スタック: （例: Node.js / Python / Go）
# ============================================================
```

#### 1-2. 視覚的マーカーの追加

各スタックブロックの前後に開始・終了マーカーを追加する:

```yaml
      # ▼ Node.js を使う場合はここからコメント解除 ▼
      # - uses: actions/setup-node@v4
      # ...
      # ▲ Node.js ここまで ▲
```

#### 1-3. PRへの自動コメントステップの追加

テスト実行ステップの後に以下を追加する:

- 使用ツール: `actions/github-script@v7`（新規アクション追加なし）
- 実行条件: `if: always()`（成功・失敗どちらでも実行）
- 重複防止: 既存コメントがあれば新規投稿ではなく**更新**する
- コメント内容:
  - ✅ または ❌ のステータス
  - 選択スタック名
  - ブランチ名
  - GitHub Actions の実行ログへのリンク

---

### 2. `docs/ci-setup.md` の新設

#### セクション1: セットアップチェックリスト

3ステップのチェックリスト形式で、最短経路でCIを動かせるよう案内する。

#### セクション2: スタック別の設定例

Node.js / Python / Go それぞれについて、`ci.yml` から対応ブロックを抜粋して掲載する。
（ファイルを行き来する手間を省く）

#### セクション3: よくある問題と対処

以下のパターンを Q&A 形式で記載する:

1. テストコマンドが見つからない（`npm test` が存在しない等）
2. 依存パッケージのインストールに失敗する
3. PRコメントが投稿されない（`permissions` 設定漏れ）
4. CIが動かない（`on:` トリガーの設定ミス）

## 触らないもの

- `ci.yml` の `concurrency` / `permissions` / `on:` トリガー（現状正しいため）
- `test-failure-issue.yml`（main失敗時のIssue起票は完成済みのため）
- 既存の3スタック（Node.js / Python / Go）以外のスタック追加（スコープ外）

---

## 成功基準

- テンプレートを初めて使う人が `docs/ci-setup.md` → `ci.yml` の順に読んで、追加説明なしにCIを動かせること
- PRを開いた状態でテスト結果（成功/失敗）と実行ログへのリンクが確認できること
